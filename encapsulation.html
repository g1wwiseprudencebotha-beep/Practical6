Encapsulation

Encapsulation is a fundamental OOP concept that bundles data (variables) and the methods (functions) that operate on that data into a single unit, known as a class.

Key Aspects of Encapsulation:

  • Hides Data: Internal representation of an objcet is hidden from the outside world.
    • Controlled Access: Access to internal data is managed via publick methods, such as getters and setters.
  • Bundling: Data and behavior is are packed together, often making the object a "black dox"
  • Improved Maintainability: Internal implementation can change without affecting internal code that uses the class.

Benefits of Encapsulation:

  • Security: Prevents unauthorised or acccidental modification of data.
  • Flexibility: Allows developers to change the internal code structure whithout breaking external, reliant code.
  • Modularity: Keeps related data and behaviors organized together.
