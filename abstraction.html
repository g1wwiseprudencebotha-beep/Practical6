Abstraction

Abstraction hides implementation details and shows only functionality to the user.It is one of the four main principles of Object-Orinted Programming.The user interacts with the object without knowing how it works internally.This makes programs easier to use, uderstand and maintain.

Real Life Examples

When you use an ATM, you insert your card, enter your PIN, and withdraw money. You do not know how the ATM connects to the bank system or processes your request. The complex internal system is hodden from. That is Abstraction.

Example:

        abstract class Animal {
            abstract void makeSound();  // abstract method
            
            void sleep() {
                System.out.printIn("Sleeping...");
            }
        }
        class Dog extends Animal {
            void makeSound() {
                System.out.printIn("Bark");
            }
        }

In this example, the Animal class is Abtsract. It contains an Abstract method called makeSound(). The Dog class inherits from the Animal and provides its own implementation of makeSound().

Why Abstraction is Important
  • Reduces code complexity
  • Improves security by hiding internal details
  • Makes code easier to maintain
  • Forces structure in programs
Difference Between Abstraction and Encapsulation

Abtsraction hides implementation details and shows only essential features. Encapsulation hides data by using private variables and controlling access through public methods.
